## 📌 이 프롬프트의 목적

아래 두 영역은 **같은 코드베이스에 존재하지만 절대 섞이면 안 된다:**

```
┌─────────────────────────────────────────────────────────────────┐
│  영역 A: Edubee CRM                                              │
│  접속자: 각 유학원 직원 / 관리자                                  │
│  URL:    [subdomain].edubee.com  또는  /  (루트)                 │
│  목적:   학생·계약·청구·비자 등 유학원 업무 전체                   │
│  포함:   CRM 메뉴 전체 + Settings (회사정보/브랜딩/도메인/유저/플랜)│
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  영역 B: Super Admin Panel                                       │
│  접속자: Edubee 운영팀 (superadmin role)만                       │
│  URL:    /superadmin/*                                          │
│  목적:   전체 테넌트 관리·플랜 관리·통계·시스템 운영              │
│  포함:   Tenants / Plans / Analytics / Billing / System         │
└─────────────────────────────────────────────────────────────────┘
```

**핵심 원칙: 기존 작동 코드를 절대 깨지 않는다**

## 🛡️ 작업 규칙

**분석 단계 (항상 먼저):**

1. 관련 파일 전체를 읽는다
1. 문제 여부를 확인한다
1. 수정 계획을 나에게 먼저 보고한다
1. **“진행해”** 라고 하기 전까지 수정하지 않는다

**수정 단계:**

- 파일 하나씩 수정 후 `npx tsc --noEmit` 확인
- 기존 정상 코드 변경 금지
- DB 컬럼 삭제/이름 변경 금지

## 🗂️ PART 1: 분리 구조 현황 분석

> 코드를 수정하기 전에 반드시 이 분석을 먼저 수행하고 결과를 보고한다.

### 분석 1-A: 프론트엔드 파일 구조 현황 파악

아래 디렉터리를 읽고 파일 트리를 출력한다:

```
/client/src/pages/
/client/src/layouts/  (또는 /client/src/components/layouts/)
/client/src/App.tsx   (또는 라우팅 파일)
```

**확인 항목:**

|확인 포인트                                                       |기대 상태  |실제 상태|
|-------------------------------------------------------------|-------|-----|
|`/pages/` 하위에 `superadmin/` 폴더가 `crm/` 또는 일반 폴더와 **분리**되어 있는가|분리됨    |□    |
|SuperAdmin 레이아웃 파일이 CRM 레이아웃 파일과 **별도** 존재하는가                |별도 존재  |□    |
|`/superadmin/*` 라우트가 CRM 라우트와 **다른 레이아웃**을 사용하는가             |별도 레이아웃|□    |
|CRM 사이드바에 `/superadmin` 링크가 **없는가**                          |없어야 함  |□    |
|SuperAdmin 사이드바에 CRM 메뉴가 **없는가**                             |없어야 함  |□    |

### 분석 1-B: 백엔드 라우트 파일 구조 현황 파악

아래 디렉터리를 읽고 파일 목록을 출력한다:

```
/server/src/routes/
/server/src/middleware/
/server/src/index.ts  (또는 app.ts)
```

**확인 항목:**

|확인 포인트                                               |기대 상태|실제 상태|
|-----------------------------------------------------|-----|-----|
|`/api/settings/*` 와 `/api/superadmin/*` 가 **별도 파일**인가|별도 파일|□    |
|`superadmin.ts` 라우트 파일에 `superAdminOnly` 미들웨어가 적용됐는가 |적용됨  |□    |
|`settings.ts` 라우트 파일에 일반 인증 미들웨어가 적용됐는가              |적용됨  |□    |
|`/api/superadmin/*` 엔드포인트가 `settings.ts`에 **없는가**    |없어야 함|□    |
|`/api/settings/*` 엔드포인트가 `superadmin.ts`에 **없는가**    |없어야 함|□    |

### 분석 1-C: API 경로 충돌 확인

`/server/src/routes/` 하위 모든 파일을 읽고 아래 패턴으로 검색한다:

```
❌ 혼재 위험 패턴 — 즉시 보고:

  /superadmin  →  잘못된 위치
  tenants      →  잘못된 위치 (테넌트 목록은 superadmin 전용)

superadmin.ts 안에:
  /settings    →  잘못된 위치
  /branding    →  잘못된 위치
  /company     →  잘못된 위치
```

### 분석 1-D: 레이아웃 컴포넌트 의존성 확인

CRM 레이아웃과 SuperAdmin 레이아웃 파일을 각각 읽고 확인한다:

```
CRM 레이아웃이 import 하는 것:
  ✅ CRM Sidebar 컴포넌트
  ✅ CRM TopNav 컴포넌트
  ❌ SuperAdminSidebar import 없어야 함
  ❌ SuperAdminNav import 없어야 함

SuperAdmin 레이아웃이 import 하는 것:
  ✅ SuperAdmin 전용 Sidebar
  ✅ SuperAdmin 전용 TopNav (또는 헤더)
  ❌ CRM Sidebar import 없어야 함
  ❌ CRM TopNav import 없어야 함
```

### 분석 1-E: 상태 관리 (Context / Store) 분리 확인

### 목표 — App.tsx 라우트 구조

```typescript
// App.tsx — 두 영역이 완전히 독립된 트리

function App() {
  return (
    <Router>
      <Routes>

        {/* ════════════════════════════════════════════ */}
        {/* 영역 A: Edubee CRM (유학원 직원용)           */}
        {/* 접근: 로그인한 모든 유저                      */}
        {/* ════════════════════════════════════════════ */}
        <Route element={<AuthGuard />}>
          <Route element={<CrmLayout />}>

            {/* CRM 메인 */}
            <Route path="/"            element={<Dashboard />} />
            <Route path="/contacts"    element={<Contacts />} />
            <Route path="/accounts"    element={<Accounts />} />
            <Route path="/leads"       element={<Leads />} />
            <Route path="/tasks"       element={<Tasks />} />
            <Route path="/quotes"      element={<Quotes />} />
            <Route path="/contracts"   element={<Contracts />} />
            <Route path="/invoices"    element={<Invoices />} />
            <Route path="/finance"     element={<Finance />} />
            <Route path="/products"    element={<Products />} />
            <Route path="/camp"        element={<Camp />} />

            {/* CRM Settings — CRM 레이아웃 안에 포함 */}
            <Route path="/settings" element={<SettingsLayout />}>
              <Route index element={<Navigate to="/settings/company" replace />} />
              <Route path="company"  element={<CompanyProfile />} />
              <Route path="branding" element={<Branding />} />
              <Route path="domain"   element={<DomainAccess />} />
              <Route path="users"    element={<UsersTeams />} />
              <Route path="plan"     element={<PlanBilling />} />
            </Route>

          </Route>
        </Route>

        {/* ════════════════════════════════════════════ */}
        {/* 영역 B: Super Admin Panel (Edubee 운영팀 전용)*/}
        {/* 접근: role === 'superadmin' 만                */}
        {/* CrmLayout 사용 안 함 — 완전 별도 레이아웃     */}
        {/* ════════════════════════════════════════════ */}
        <Route element={<SuperAdminGuard />}>
          <Route element={<SuperAdminLayout />}>  {/* ← CrmLayout 아님 */}

            <Route path="/superadmin"
                   element={<SuperAdminDashboard />} />
            <Route path="/superadmin/tenants"
                   element={<TenantList />} />
            <Route path="/superadmin/tenants/:id"
                   element={<TenantDetail />} />
            <Route path="/superadmin/plans"
                   element={<PlatformPlans />} />

          </Route>
        </Route>

        {/* ════════════════════════════════════════════ */}
        {/* 공용 페이지 (레이아웃 없음)                   */}
        {/* ════════════════════════════════════════════ */}
        <Route path="/login"  element={<Login />} />
        <Route path="/404"    element={<NotFound />} />
        <Route path="*"       element={<Navigate to="/404" replace />} />

      </Routes>
    </Router>
  );
}
```

### 목표 — 두 영역의 완전 독립 체크포인트

```
독립성 체크 (이 중 하나라도 위반되면 즉시 수정):

프론트엔드
  □ CrmLayout 은 SuperAdminLayout 을 import 하지 않는다
  □ SuperAdminLayout 은 CrmLayout 을 import 하지 않는다
  □ CRM 사이드바에 /superadmin 링크가 없다
  □ SuperAdmin 사이드바에 /settings, /contacts 등 CRM 링크가 없다
  □ CRM 페이지에서 useAuth().role === 'superadmin' 체크로 분기하지 않는다
      (SuperAdmin 기능은 SuperAdmin 페이지에서만 노출)

백엔드
  □ settings.ts 에 superAdminOnly 미들웨어가 없다
      (설정 API는 일반 인증으로만 보호)
  □ superadmin.ts 의 모든 라우트에 superAdminOnly 가 있다
  □ settings.ts 에 /tenants, /platform-plans 엔드포인트가 없다
  □ superadmin.ts 에 /company, /branding, /domain 엔드포인트가 없다

DB 접근
  □ superadmin 라우트는 organisation_id 필터 없이 전체 조회 가능
      (모든 테넌트를 봐야 하므로 — 의도적 예외)
  □ settings 라우트는 반드시 organisation_id 필터 적용
      (자기 테넌트 데이터만 접근)
```

### Separation Fix 1: CrmLayout — SuperAdmin 완전 분리

**조건:** CrmLayout 과 SuperAdminLayout 이 공유하는 코드가 있거나 한쪽이 다른 쪽을 import 하는 경우

**작업 내용:**

두 레이아웃은 아래처럼 **완전히 독립된 파일**이어야 한다.

```typescript
// CrmLayout.tsx — CRM 전용
// import 에 superadmin 관련 내용 없음
export function CrmLayout() {
  return (
    <div style={{ display: 'flex', height: '100vh' }}>
      <CrmSidebar />                {/* CRM 전용 사이드바 */}
      <div style={{ flex: 1, display: 'flex', flexDirection: 'column' }}>
        <CrmTopNav />               {/* CRM 전용 탑바 */}
        <main>
          <Outlet />                {/* CRM 페이지들 렌더링 */}
        </main>
      </div>
    </div>
  );
}
```

```typescript
// SuperAdminLayout.tsx — Super Admin 전용
// import 에 CRM 관련 내용 없음
export function SuperAdminLayout() {
  return (
    <div style={{ display: 'flex', height: '100vh' }}>
      <SuperAdminSidebar />         {/* Super Admin 전용 사이드바 */}
      <div style={{ flex: 1, display: 'flex', flexDirection: 'column' }}>
        <SuperAdminTopNav />        {/* Super Admin 전용 헤더 */}
        <main>
          <Outlet />                {/* Super Admin 페이지들 렌더링 */}

      </div>

```

### Separation Fix 2: CRM 사이드바 — SuperAdmin 링크 없음 확인

`/client/src/components/crm/Sidebar.tsx` (또는 해당 파일)을 읽고:

```

  /superadmin 링크

  "Admin Panel" 같은 운영자 전용 메뉴

✅ 있어야 할 것:
  CRM 메뉴 (Dashboard, Contacts, Leads 등)
  Settings 섹션 (Company, Branding, Domain, Users, Plan)
```

### Separation Fix 4: SuperAdminLayout 디자인 차별화

Super Admin은 일반 유학원 직원이 보면 안 되는 운영 영역이다.
CRM과 시각적으로도 명확히 구분되어야 한다.

```
CRM 영역 디자인:
  Primary:  #F5821F (Edubee Orange)
  배경:     #FAFAF9
  사이드바: #F4F3F1

Super Admin 영역 디자인 (차별화):
  헤더 배경:      #1C1917  (Neutral 900 — 다크 헤더)
  "SUPER ADMIN" 배지: 빨간 배지 (#DC2626)
  사이드바:       #FAFAF9  (동일해도 무방)
  탑바에 경고 배너: "⚠️ Super Admin Mode — 운영 환경입니다"
```

`SuperAdminLayout.tsx` 상단에 다음을 반드시 포함한다:

```typescript
// Super Admin 전용 경고 배너
```

### Separation Fix 5: 백엔드 라우트 파일 혼재 해소

`/server/src/routes/settings.ts` 와 `/server/src/routes/superadmin.ts` 를 각각 읽고,
**엔드포인트가 잘못된 파일에 있으면 올바른 파일로 이동**한다.

**이동 기준:**

|엔드포인트              |있어야 할 파일       |
|-------------------|---------------|
|`/api/settings/*`  |`settings.ts`  |
|`/api/superadmin/*`|`superadmin.ts`|
|테넌트 생성/수정/삭제       |`superadmin.ts`|
|플랜 마스터 CRUD        |`superadmin.ts`|
|회사정보/브랜딩/도메인       |`settings.ts`  |
|직원 초대              |`settings.ts`  |


> ⚠️ 엔드포인트 이동 시 기존 URL은 변경하지 않는다.
> 단순히 어떤 파일에 정의됐는지의 문제이며
> `index.ts` 등록 경로가 맞으면 동작은 동일하다.

### Separation Fix 6: index.ts 라우트 등록 순서 및 구분 주석

`/server/src/index.ts` 를 읽고 라우트 등록 부분이 명확히 구분되어 있는지 확인한다.
구분 주석이 없거나 순서가 잘못됐으면 아래처럼 정리한다:

```typescript
// ──────────────────────────────────────────────────────
// 1. 공용 미들웨어
// ──────────────────────────────────────────────────────
app.use(cors(...));
app.use(express.json());

// ──────────────────────────────────────────────────────
// 2. 인증 미들웨어 (순서 중요 — 아래 라우트들보다 먼저)
// ──────────────────────────────────────────────────────
app.use(authenticate);      // 토큰 검증 → req.user 주입
app.use(tenantResolver);    // 테넌트 식별 → req.tenantId 주입

// ──────────────────────────────────────────────────────
// 3. 공용 라우트 (인증 불필요)
// ──────────────────────────────────────────────────────
app.use('/api/auth', authRoutes);

// ──────────────────────────────────────────────────────
// 4. CRM 라우트 (일반 인증 필요)
//    접근: 로그인한 모든 유저
// ──────────────────────────────────────────────────────
app.use('/api/contacts',  contactsRoutes);
app.use('/api/accounts',  accountsRoutes);
app.use('/api/leads',     leadsRoutes);
app.use('/api/tasks',     tasksRoutes);
app.use('/api/quotes',    quotesRoutes);
app.use('/api/contracts', contractsRoutes);
app.use('/api/invoices',  invoicesRoutes);
app.use('/api/products',  productsRoutes);
app.use('/api/settings',  settingsRoutes);  // CRM Settings

// ──────────────────────────────────────────────────────
// 5. Super Admin 라우트 (superadmin role 필요)
//    접근: role === 'superadmin' 만
//    superAdminOnly 미들웨어는 superadmin.ts 내부에 적용됨
// ──────────────────────────────────────────────────────
app.use('/api/superadmin', superAdminRoutes);
```

## ✅ PART 4: 분리 검증 체크리스트

모든 분석 및 수정 완료 후 아래 항목을 확인한다.

### 프론트엔드 분리 검증

```
레이아웃
  □ CrmLayout.tsx 에 SuperAdminLayout import 없음
  □ SuperAdminLayout.tsx 에 CrmLayout import 없음
  □ 두 레이아웃이 공유하는 컴포넌트 없음
    (공유가 필요한 경우 /components/shared/ 에 명시적으로 분리)

라우팅
  □ /superadmin/* 라우트가 SuperAdminGuard 로 감싸져 있음
  □ /settings/* 라우트가 CrmLayout 안에 있음
  □ App.tsx 에서 두 트리가 완전히 분리됨

사이드바
  □ CRM 사이드바에 /superadmin 링크 없음
  □ SuperAdmin 사이드바에 CRM 메뉴 링크 없음

시각적 구분
  □ SuperAdminLayout 에 빨간 경고 배너 있음
  □ SuperAdmin 헤더에 "SUPER ADMIN" 배지 있음
```

### 백엔드 분리 검증

```
파일 구조
  □ settings.ts 에 /superadmin 패턴 엔드포인트 없음
  □ superadmin.ts 에 /settings 패턴 엔드포인트 없음

미들웨어 적용
  □ superadmin.ts 의 모든 router.XXX() 에 superAdminOnly 적용됨
  □ settings.ts 에 superAdminOnly 미들웨어 없음

index.ts 등록
  □ 구분 주석으로 CRM / Super Admin 영역 명확히 분리됨
  □ authenticate → tenantResolver 순서 유지됨
  □ /api/superadmin 이 /api/settings 와 별도 경로로 등록됨
```

### DB 접근 패턴 검증

```
CRM Settings API (settings.ts)
  □ 모든 SELECT에 WHERE organisation_id = req.tenantId 조건 있음
  □ 다른 테넌트 데이터 절대 접근 불가

Super Admin API (superadmin.ts)
  □ 전체 테넌트 조회 시 organisation_id 필터 없음 (의도적)
  □ 단, req.user.role === 'superadmin' 검증 완료 후에만 실행됨
```
