⚠️ REPLIT ENVIRONMENT NOTES (read before coding):
- Do NOT use local filesystem for temp files (ephemeral storage resets).
  All CSV generation must happen IN MEMORY (Buffer/stream), 
  never write to disk with fs.writeFile.
- Use streaming responses for large exports to avoid memory overflow.
- Replit has ~512MB RAM limit — chunk large DB queries 
  (use LIMIT/OFFSET pagination, max 1000 rows per query batch).
- Install new packages via package.json only, not shell commands.
  Add to dependencies: "csv-parse": "^5.5.0", "csv-stringify": "^6.4.0"
- Do NOT use child_process or worker_threads (restricted in Replit).
- All file downloads must use res.setHeader() + res.send(buffer),
  NOT res.sendFile() or fs streams.
- Connection pool: keep pg pool max at 5 (Replit DB connection limit).
- Timeout: set express timeout to 30s for export routes 
  (large exports may be slow).
- CORS: already configured — do not add new cors() middleware.
- Environment variables: access only via process.env, 
  never hardcode credentials.

 PART 5: IMPORT API
═══════════════════════════════════════════

POST /api/data-manager/import/:tableName
  Content-Type: multipart/form-data
  Field: file (CSV)

⚠️ REPLIT UPLOAD RULES:
  - Use multer memoryStorage() — NEVER diskStorage()
  - Max file size: 5MB (set in multer limits)
  - Parse from req.file.buffer directly using csv-parse
  - Process in batches of 100 rows per DB transaction
  - If batch fails, rollback that batch only (partial success ok)

IMPORT FLOW:
  1. Parse CSV from buffer (csv-parse, skip #EXAMPLE rows)
  2. Validate each row (type check, required fields, format)
  3. Collect errors: [{ row, field, message }]
  4. Insert valid rows in batches of 100
  5. Save to import_history table
  6. Return: {
       totalRows, successRows, errorRows,
       importHistoryId,
       errorCsv: base64-encoded CSV of failed rows with error column
     }

IMPORT-ENABLED TABLES & VALIDATION RULES:

users:
  Required: email*, role*, full_name*
  Validate: email format, role must be valid enum,
            no duplicate email (skip+error if exists)
  Auto-set: status='pending', password=temp random (send email)
  ⚠️ Never allow import of super_admin role

exchange_rates:
  Required: from_currency*, rate*, effective_date*
  Validate: rate > 0, date format YYYY-MM-DD,
            currency must be in allowed list
            [KRW,THB,JPY,USD,PHP,SGD,GBP,AUD]
  Behavior: UPSERT on (from_currency, to_currency, effective_date)

products:
  Required: product_name*, product_type*, cost*
  Validate: product_type in [institute,hotel,pickup,tour,settlement]
            cost is numeric, cost > 0

enrollment_spots:
  Required: package_group_id*, grade_label*, total_spots*
  Validate: package_group_id exists, total_spots integer > 0
  Behavior: UPSERT on (package_group_id, grade_label)

transactions:
  Required: contract_id*, transaction_type*, amount*,
            transaction_date*
  Validate: contract_id exists, amount numeric,
            date format YYYY-MM-DD
  Auto-set: created_by = current user id
