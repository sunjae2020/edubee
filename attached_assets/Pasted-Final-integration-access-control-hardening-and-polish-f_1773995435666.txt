 PART 1: ACCESS CONTROL AUDIT
═══════════════════════════════════════════

Verify and enforce these rules across ALL report endpoints and pages:

BACKEND — add to each route handler:

  GET /api/reports / GET /api/reports/:id:
    parent_client → status must be 'published' AND
                    application.client_id = req.user.id
    education_agent → status must be 'published' AND
                      application.agent_id = req.user.id
    camp_coordinator → contract.camp_provider_id = req.user.id
                       (draft OR published)
    super_admin / admin → unrestricted

  POST /api/reports:
    parent_client → 403 always
    education_agent → 403 always
    camp_coordinator → contract.camp_provider_id must = req.user.id

  PATCH /api/reports/:id/unpublish:
    camp_coordinator → 403 always
    education_agent → 403 always
    parent_client → 403 always
    Only super_admin and admin allowed

  DELETE /api/reports/:id:
    Only super_admin allowed (admin → 403)

  GET /api/reports/:id/pdf:
    Same rules as GET /:id
    Additionally: if status='draft' AND role is EA or parent → 403
    Passport fields: strip passportNumber + passportExpiry
    from section content BEFORE PDF generation
    if role is education_agent or parent_client

FRONTEND — route guards:

  /admin/reports/:id/edit:
    If effectiveRole is 'education_agent' or 'parent_client'
    → redirect to /admin/reports/:id

  /admin/reports:
    If effectiveRole is 'parent_client'
    → redirect to /admin/my-programs

  Report viewer — field visibility:
    If effectiveRole in ['education_agent', 'parent_client']:
      passport number field → show "— (Hidden)"
      passport expiry field → show "— (Hidden)"
