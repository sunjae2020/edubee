## 작업 개요
files 테이블에 문서 카테고리 관리 기능을 추가한다.
학생 공식 서류와 상담 업무 파일을 구분하기 위한 컬럼 3개를 추가하고,
관련 백엔드 API와 프론트엔드 UI를 함께 구현한다.

---

## 작업 전 필수 확인 (수정 전 반드시 읽고 보고할 것)

아래 파일들을 먼저 전부 읽어라:
1. `/db/schema.ts` → files 테이블 Drizzle 스키마 확인
2. `/server/src/routes/` → files 관련 라우트 파일 확인
3. `/client/src/pages/` → 파일 업로드/목록 관련 페이지 확인
4. `/client/src/components/` → FileUpload 또는 파일 관련 컴포넌트 확인

읽은 후 수정할 파일 목록과 변경 내용을 먼저 보고하고,
내가 "진행해"라고 할 때까지 코드를 수정하지 마라.

## 구현 요구사항

### 1단계 — DB 마이그레이션

`files` 테이블에 아래 컬럼 3개를 추가한다.
```sql
document_category  VARCHAR(50)    NULL   -- 카테고리 코드 (아래 목록 참조)
expires_at         TIMESTAMP      NULL   -- 만료일 (여권, 잔고증명 등)
is_submitted       BOOLEAN        NULL   DEFAULT false  -- 학교/기관 제출 여부
submitted_to       VARCHAR(200)   NULL   -- 제출 대상 (학교명, 이민성 등)
```

`document_category` 허용 값 목록 (CHECK constraint 또는 코드 레벨에서 검증):
학생 신분 서류 그룹:
PASSPORT        여권
PHOTO_ID        사진 / 신분증
ACADEMIC        성적증명서 / 졸업증명서
ENGLISH_TEST    IELTS / TOEFL / PTE
FINANCIAL       은행잔고 / 재정보증서
비자 / 입학 서류 그룹:
VISA_DOC        비자 신청·승인·거절 서류
COE             Confirmation of Enrolment
OFFER_LETTER    LOO / Offer Letter
INSURANCE       해외 유학생 보험
상담 업무 파일 그룹:
CONSULTATION    상담신청서 / Jotform
QUOTATION       견적서 PDF
CONTRACT_DOC    계약서 파일
CORRESPONDENCE  이메일 / 메모 / 공문
SCHOOL_INFO     학교 브로셔 / 안내자료
OTHER           기타

`/db/schema.ts` Drizzle 스키마에도 동일하게 반영한다.
마이그레이션은 `npx drizzle-kit push` 로 실행한다.

### 2단계 — 백엔드 API

기존 files 라우트(CRUD)에 아래 사항을 추가/수정한다.
기존에 작동하는 코드는 절대 건드리지 않는다.

**추가할 기능:**

(A) 파일 업로드 API (`POST /api/files`) — 기존 필드에 아래 추가 수신:
  - `document_category` (string)
  - `expires_at` (ISO 날짜 string, optional)
  - `is_submitted` (boolean, optional)
  - `submitted_to` (string, optional)

(B) 파일 목록 조회 API (`GET /api/files`) — 쿼리 파라미터 추가:
  - `entity_type` (기존)
  - `entity_id` (기존)
  - `document_category` (신규 필터)
  - `group` 파라미터: `student_docs` 또는 `consultation_docs`
    - `student_docs` → PASSPORT, PHOTO_ID, ACADEMIC, ENGLISH_TEST, FINANCIAL, VISA_DOC, COE, OFFER_LETTER, INSURANCE 만 반환
    - `consultation_docs` → CONSULTATION, QUOTATION, CONTRACT_DOC, CORRESPONDENCE, SCHOOL_INFO, OTHER 만 반환

(C) 만료 임박 서류 조회 API (`GET /api/files/expiring`):
  - `days` 파라미터 (default: 30) — N일 이내 만료 예정 파일 반환
  - `expires_at IS NOT NULL AND expires_at <= NOW() + INTERVAL 'N days'` 조건
  - 반환 필드: id, name, document_category, expires_at, entity_type, entity_id, contact 이름

### 3단계 — 프론트엔드 UI

학생 상세 페이지(Contact 또는 Account 상세)의 파일 섹션을 두 탭으로 분리한다.

**탭 구성:**

📁 학생 서류 탭 (student_docs 그룹)
  - 카테고리별 섹션으로 그룹핑해서 표시
  - 만료일이 있는 항목은 D-day 뱃지 표시:
    - 만료됨: 빨간색 "만료됨" 뱃지
    - 30일 이내: 주황색 "D-N" 뱃지
    - 정상: 초록색 날짜 표시
  - 파일 업로드 시 document_category 드롭다운 필수 선택 (학생 서류 그룹만 표시)

📎 상담/계약 파일 탭 (consultation_docs 그룹)
  - 날짜순 목록으로 표시 (만료일 배지 불필요)
  - 파일 업로드 시 document_category 드롭다운 (상담 업무 파일 그룹만 표시)

**공통 업로드 모달 수정:**
- `document_category` 드롭다운 추가 (필수 입력)
- `expires_at` 날짜 입력 (document_category가 PASSPORT, FINANCIAL, ENGLISH_TEST, VISA_DOC일 때만 표시)
- `submitted_to` 텍스트 입력 (VISA_DOC, COE, OFFER_LETTER일 때만 표시)
