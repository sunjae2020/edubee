── ROLE VIEW SWITCHER ────────────────────────────────────────

Show ONLY for: super_admin, admin, camp_coordinator

ROLE_HIERARCHY: super_admin:100 | admin:80 | camp_coordinator:60
  education_agent:40 | partner_*:30 | parent_client:20

canSwitchTo(my, target) = HIERARCHY[my] > HIERARCHY[target]
  → NEVER allow upward switch

Normal button: [🔍 View as: [My Role] ▾]

Dropdown shows users with lower hierarchy:
  Roles with multiple users → sub-menu per user
  Fetch from GET /api/users/switchable

Impersonating button: [👁 Viewing as: Sarah Kim ▾] (amber)
  Dropdown adds: [↩ Return to My Account] at top

Behavior:
  Sidebar filtered by effectiveRole
  All API calls send X-View-As-User-Id header
  Backend partnerScope uses effectiveUserId for data filtering
  Stored in sessionStorage → clears on tab close
  All sessions logged to impersonation_logs

Permissions table:
  super_admin → can view as: all lower roles
  admin       → can view as: CC, EA, partners, parent (NOT SA)
  camp_coord  → can view as: EA, partners, parent (NOT SA/AD/CC)
  Others      → ✗ not shown

  KPIs: My Applications | Approved | Commission MTD
  Tables: Recent applications | My clients

partner_* (own assignments only):
  Upcoming assignments next 7 days | Assignment list

parent_client:
  Program status card (large) | Service timeline (✈→🎓→🏨→🗺️)
  Payment status | [View Full Report] if published

═══════════════════════════════════════════
 PART 6: APPLICATIONS (/admin/applications)
═══════════════════════════════════════════

Data scope by role:
  SA/AD: all applications
  camp_coordinator: WHERE packageGroup.camp_provider_id = own id
  education_agent: WHERE agent_id = own id
  parent_client: WHERE client_id = own id

List filters: Status | Agent | Program | Date range
Status badges: pending=yellow | under_review=blue |
  interview_scheduled=purple | approved=green |
  rejected=red | contracted=indigo | cancelled=gray

Application Detail TABS:

[Overview]: Details + Status timeline
  SA/AD/CC actions: [Approve] [Reject] [Convert to Contract] [Schedule Interview]
  EA: readonly status

[Participants]:
  Primary Student card (all info, age auto-calc)
  Additional Children cards | [Add Child]
  Accompanying Adults cards | [Add Adult]
  Bar: "👦 2  👨 1  Total: 3"

[Interview] (if package requires):
  Not scheduled: [Schedule Interview]
  Scheduled: info card + result section (CC/admin only)

[Documents]: (LEFT: checklist | RIGHT: cards + upload)
  ✅ Approved | ⏳ Pending | ⬜ Optional | ❌ Rejected | ⚠️ Expiring
  Upload modal: drag&drop | Category | Participant link | Expiry | Notes

[Notes]: Internal log (CC/admin only) + [Add Note]

POST /api/applications/:id/convert-contract
  → Creates contract + contractNumber (CNT-2026-XXXX)
  → Creates empty service records (institute/hotel/pickup/tour/settlement)
  → Returns { contractId }
