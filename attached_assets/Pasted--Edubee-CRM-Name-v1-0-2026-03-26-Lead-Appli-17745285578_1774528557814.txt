## 1. 이름 필드 정의

### 1.1 필드 4종 정의

| 필드명 | 표시명 (KO) | 표시명 (EN) | 설명 | 예시 |
|--------|------------|------------|------|------|
| `first_name` | 영문 이름 | First Name | 영문 이름 (Given name) | `Jason` |
| `last_name` | 영문 성 | Last Name | 영문 성 (Family name) | `Kim` |
| `english_name` | 영문 별명 | English Name / Nickname | 선호 영문 닉네임 (선택) | `Jay` |
| `original_name` | 한국어 이름 | Original Name | 모국어 이름 (한국어·일본어·중국어 등) | `김민준` |

### 1.2 파생 조합 규칙 (Computed — DB에 저장 안 함)

| 조합명 | 공식 | 예시 | 사용처 |
|--------|------|------|--------|
| `fullName` | `{first_name} {LAST_NAME}` | `Jason KIM` | 모든 모듈 기본 표시 |
| `displayName` | original_name이 있으면 `{fullName} ({original_name})`, 없으면 `{fullName}` | `Jason KIM (김민준)` | 리스트 검색, 드롭다운 |
| `accountName` | Account 생성 시 `name` 필드 → `{first_name} {LAST_NAME}` | `Jason KIM` | accounts.name |
| `shortName` | `{first_name} {last_name[0]}.` | `Jason K.` | 모바일/컴팩트 뷰 |

### 1.3 표시 규칙

```
UI 표시 순서 (2줄 표기):

  Jason KIM       ← first_name + LAST_NAME (14px/600, color: #1C1917, Account 링크)
  김민준          ← original_name (12px, color: #57534E, 없으면 생략)
```

> **LAST_NAME 대문자 규칙**: `last_name`은 항상 `toUpperCase()`로 표시  
> **원본 저장**: DB에는 원래 입력값(예: `kim`) 그대로 저장 → 표시 시 변환

## 2. DB 스키마 추가

> ⚠️ **절대 컬럼 삭제/이름 변경 금지. ADD COLUMN만 수행.**

### 2.1 contacts 테이블 추가

```sql
-- contacts 테이블: english_name, original_name 추가
-- ※ other_name은 기존 필드 유지 (하위 호환)
ALTER TABLE contacts
  ADD COLUMN IF NOT EXISTS english_name   VARCHAR(100),   -- 영문 닉네임 (선택)
  ADD COLUMN IF NOT EXISTS original_name  VARCHAR(200);   -- 모국어 이름 (한국어 등)

-- 인덱스 (검색 성능)
CREATE INDEX IF NOT EXISTS idx_contacts_original_name ON contacts(original_name);
CREATE INDEX IF NOT EXISTS idx_contacts_full_name ON contacts(first_name, last_name);
```

### 2.5 applications 테이블 신규 생성 (일반 유학·비자 신청)

> ⚠️ **현재 스키마에 없는 신규 테이블. `camp_applications`와 별개로 운영.**  
> 일반 유학 상담 신청 (랜딩 페이지 또는 직접 접수) → Lead 자동 생성 진입점.

```sql
  id                    UUID        PRIMARY KEY DEFAULT gen_random_uuid(),

  last_name             VARCHAR(100) NOT NULL,     -- 영문 성
  english_name          VARCHAR(100),              -- 영문 닉네임 (선택)
  original_name         VARCHAR(200),              -- 모국어 이름 (한국어 등, 선택)

  phone                 VARCHAR(50),
  nationality           VARCHAR(100),
  date_of_birth         DATE,
  gender                VARCHAR(20),               -- Male | Female | Other

  -- 신청 내용
  inquiry_type          VARCHAR(100),              -- Study Abroad | Visa | Working Holiday | Language School | Internship | Other
  preferred_country     VARCHAR(100),
  preferred_course      VARCHAR(255),
  preferred_start_date  DATE,
  budget_aud            DECIMAL(12,2),
  study_duration_weeks  INTEGER,

  -- 추가 정보
  current_english_level VARCHAR(50),              -- Beginner | Elementary | Intermediate | Upper-Intermediate | Advanced
  has_passport          BOOLEAN,
  passport_expiry_date  DATE,
  special_requirements  TEXT,
  how_did_you_hear      VARCHAR(100),             -- Online Ads | Referral | SNS | Walk-in | Other

  -- SNS
  sns_type              VARCHAR(50),              -- LINE | WeChat | KakaoTalk | WhatsApp
  sns_id                VARCHAR(100),

  -- CRM 연동 (확정 후)
  lead_id               UUID        REFERENCES leads(id),      -- 자동 생성
  contact_id            UUID        REFERENCES contacts(id),   -- 확정 후 연결

  assigned_staff_id     UUID        REFERENCES users(id),

  utm_source            VARCHAR(100),
  utm_campaign          VARCHAR(100),
  ip_address            VARCHAR(50),

  -- 상태
  application_status    VARCHAR(50) NOT NULL DEFAULT 'submitted',
  -- submitted | reviewing | qualified | converted | cancelled
  reviewed_at           TIMESTAMP,
  converted_at          TIMESTAMP,                -- Lead/Account 변환 시각
  cancellation_reason   TEXT,

  created_at            TIMESTAMP   NOT NULL DEFAULT NOW(),
  updated_at            TIMESTAMP   NOT NULL DEFAULT NOW()

-- 인덱스
CREATE INDEX IF NOT EXISTS idx_applications_email         ON applications(email);
CREATE INDEX IF NOT EXISTS idx_applications_status        ON applications(application_status);
CREATE INDEX IF NOT EXISTS idx_applications_original_name ON applications(original_name);
CREATE INDEX IF NOT EXISTS idx_applications_full_name     ON applications(first_name, last_name);
CREATE INDEX IF NOT EXISTS idx_applications_lead_id       ON applications(lead_id);
```

**채번 규칙:** `APP-XXXXXXXX` (8자리 순번, camp의 `CAMP-` 와 구분)

**Application Status 흐름:**
```
submitted → reviewing → qualified → converted (→ Lead/Account 생성)

```

## 3. TypeScript 유틸리티 함수

> 파일 위치: `client/src/lib/nameUtils.ts`

```typescript
// ============================================================
// Edubee CRM — Name Utility Functions
// 모든 이름 관련 로직은 이 파일에서 중앙 관리
// ============================================================

export interface NameFields {
  firstName?:    string | null;
  lastName?:     string | null;
  englishName?:  string | null;
  originalName?: string | null;
}

/**
 * fullName: "Jason KIM" 형태
 * - lastName은 항상 UPPERCASE
 * - firstName만 있으면 firstName만 반환
 * - 둘 다 없으면 fallback 반환
 */
export function buildFullName(
  fields: NameFields,
  fallback = "—"
): string {
  const first = fields.firstName?.trim() || "";
  const last  = fields.lastName?.trim().toUpperCase() || "";
  if (first && last) return `${first} ${last}`;
  if (first)         return first;
  if (last)          return last;
  return fallback;
}

/**
 * displayName: 검색/드롭다운용
 * 원본 이름 있으면 → "Jason KIM (김민준)"
 * 없으면           → "Jason KIM"
 */
export function buildDisplayName(
  fields: NameFields,
  fallback = "—"
): string {
  const full     = buildFullName(fields, "");
  const original = fields.originalName?.trim() || "";
  if (full && original) return `${full} (${original})`;
  if (full)             return full;
  if (original)         return original;
  return fallback;
}

/**
 * accountName: Account.name 필드에 저장할 값
 * → fullName 기준 (영문 표준명)
 */
export function buildAccountName(fields: NameFields): string {
  return buildFullName(fields, "Unknown");
}

/**
 * shortName: 컴팩트 뷰용 "Jason K."
 */
export function buildShortName(
  fields: NameFields,
  fallback = "—"
): string {
  const first = fields.firstName?.trim() || "";
  const last  = fields.lastName?.trim() || "";
  if (first && last) return `${first} ${last[0].toUpperCase()}.`;
  return buildFullName(fields, fallback);
}

/**
 * nameFromAccount: Account 레코드에서 NameFields 추출
 * Account.name이 단일 문자열인 경우에도 처리
 */
export function nameFromAccount(account: {
  firstName?: string | null;
  lastName?:  string | null;
  englishName?: string | null;
  originalName?: string | null;
  name?: string;
}): NameFields {
  // 분할 필드가 있으면 우선 사용
  if (account.firstName || account.lastName) {
    return {
      firstName:    account.firstName,
      lastName:     account.lastName,
      englishName:  account.englishName,
      originalName: account.originalName,
    };
  }
  // fallback: name 단일 필드 → firstName으로 처리
  return { firstName: account.name };
}

/**
 * nameFromContact: Contact 레코드에서 NameFields 추출
 */
export function nameFromContact(contact: {
  firstName?: string | null;
  lastName?:  string | null;
  englishName?: string | null;
  originalName?: string | null;
  otherName?: string | null;   // 기존 other_name → original_name fallback
}): NameFields {
  return {
    firstName:    contact.firstName,
    lastName:     contact.lastName,
    englishName:  contact.englishName,
    originalName: contact.originalName || contact.otherName, // 하위 호환
  };
}

/**
 * nameFromApplication: 일반 Application 레코드에서 NameFields 추출
 * applications 테이블은 처음부터 4분할 필드로 설계됨
 */
export function nameFromApplication(application: {
  firstName?:    string | null;
  lastName?:     string | null;
  englishName?:  string | null;
  originalName?: string | null;
}): NameFields {
  return {
    firstName:    application.firstName,
    lastName:     application.lastName,
    englishName:  application.englishName,
    originalName: application.originalName,
  };
}

/**
 * nameFromCampApplication: Camp Application 레코드에서 NameFields 추출
 * applicant_ prefix 필드 → 공통 NameFields로 정규화
 */
export function nameFromCampApplication(app: {
  applicantFirstName?:    string | null;
  applicantLastName?:     string | null;
  applicantEnglishName?:  string | null;
  applicantOriginalName?: string | null;
  applicantName?:         string | null;  // 기존 단일 필드 fallback
}): NameFields {
  if (app.applicantFirstName || app.applicantLastName) {
    return {
      firstName:    app.applicantFirstName,
      lastName:     app.applicantLastName,
      englishName:  app.applicantEnglishName,
      originalName: app.applicantOriginalName,
    };
  }
  // fallback: 기존 applicantName 단일 필드
  return { firstName: app.applicantName };
}
```

### 4.1 ClientNameDisplay (2줄 표시 — 링크 있음)

```tsx
// ============================================================
// <ClientNameDisplay />
// Quote, Contract, Service 모듈에서 공통 사용
//
// 표시 예:
//   김민준         ← originalName (small, grey, 없으면 생략)
//   Jason KIM →    ← fullName (bold, orange link → Account 페이지)
// ============================================================
import { Link } from "wouter"; // 또는 react-router-dom
import { buildFullName } from "@/lib/nameUtils";
import type { NameFields } from "@/lib/nameUtils";

interface ClientNameDisplayProps {
  fields:     NameFields;
  accountId?: string | null;   // 링크 대상 Account ID
  size?:      "sm" | "md" | "lg";
  className?: string;
}

export function ClientNameDisplay({
  fields,
  accountId,
  size = "md",
  className = "",
}: ClientNameDisplayProps) {
  const fullName     = buildFullName(fields);
  const originalName = fields.originalName?.trim();
  const englishName  = fields.englishName?.trim();

  const fontSizes = {
    sm: { original: "text-[11px]", full: "text-xs",  english: "text-[11px]" },
    md: { original: "text-xs",     full: "text-sm",  english: "text-xs"     },
    lg: { original: "text-sm",     full: "text-base",english: "text-sm"     },
  };
  const fs = fontSizes[size];

  return (
    <div className={`flex flex-col gap-0.5 ${className}`}>
      {/* Full Name — Account 링크 (항상 첫 번째) */}
      {accountId ? (
        <Link
          href={`/accounts/${accountId}`}
          className={`${fs.full} font-semibold text-[#F5821F] hover:text-[#D96A0A]
                      hover:underline transition-colors leading-tight`}
        >
          {fullName}
        </Link>
      ) : (
        <span className={`${fs.full} font-semibold text-[#1C1917] leading-tight`}>
          {fullName}
        </span>
      )}

      {/* Original Name (모국어) — 있을 때만 두 번째 줄 표시 */}
      {originalName && (
        <span
          className={`${fs.original} text-[#57534E] leading-tight`}
          title="Original Name"
        >
          {originalName}
        </span>
      )}

      {/* English Nickname — 있을 때만 표시 (회색 괄호) */}
      {englishName && englishName !== fields.firstName && (
        <span className={`${fs.english} text-[#A8A29E] leading-tight`}>
          ({englishName})
        </span>
      )}
    </div>
  );
}
```

### 4.2 ClientNameCell (테이블 셀 전용 — 1줄 컴팩트)

```tsx
// ============================================================
// <ClientNameCell />

  const fullName     = buildFullName(fields);
  const originalName = fields.originalName?.trim();

  return (
    <div className="flex flex-col gap-0.5 py-1">
      {/* 메인 이름 (fullName 먼저) */}
      <div
        className="flex items-center gap-1.5"

          </Link>
```

## 6. Account / Contact 변환 흐름

### 6.1 변환 시점 정의

**3가지 진입 경로 모두 동일한 변환 흐름을 따른다:**

```
┌──────────────────────────────────────────────────────────────┐
│  진입 경로 A: Lead 직접 생성 (직원이 CRM에서 직접 입력)          │
│  진입 경로 B: Application 제출 (일반 유학·비자 신청 랜딩 페이지) │
│  진입 경로 C: Camp Application 제출 (캠프 랜딩 페이지)          │
└──────────────────────────────────────────────────────────────┘
     ↓ 최초 신청 정보 수집 (4개 이름 필드 공통 포함)

[담당자 검토 단계]
     ↓ 상담 진행
     ↓ application_status: submitted → reviewing → qualified

[Account & Contact 생성 트리거]
  - 경로 A (Lead): "Convert to Account" 버튼 클릭 시
  - 경로 B (Application): application_status → "converted" 시
  - 경로 C (Camp Application): application_status → "confirmed" 시

     ↓ "Account & Contact 생성" 처리
     
[Account 생성]                    [Contact 생성]
  accounts.name       = fullName    contacts.first_name    = firstName
  accounts.first_name = firstName   contacts.last_name     = lastName
  accounts.last_name  = lastName    contacts.english_name  = englishName
  accounts.english_name = english   contacts.original_name = originalName
  accounts.original_name = original contacts.account_id    = account.id
  accounts.account_type = 'Student'
     ↓
[각 소스 레코드 → account_id, contact_id 연결]
  leads.account_id              = account.id
  leads.contact_id              = contact.id
  applications.account_id       = account.id   ← 일반 Application
  applications.contact_id       = contact.id
  camp_applications.account_id  = account.id   ← Camp (간접: lead 통해)
```

```typescript
// ============================================================
// Account/Contact 생성 헬퍼
// ============================================================

interface SourceNameFields {
  firstName?:    string | null;
  lastName?:     string | null;
  englishName?:  string | null;
  originalName?: string | null;
}

/**
 * Lead/Application → Account 생성 payload 생성
 */
export function buildAccountFromName(
  fields: SourceNameFields,
  extra: {
    email?:   string;
    phone?:   string;
    ownerId:  string;
  }
) {
  const first = fields.firstName?.trim() || "";
  const last  = (fields.lastName?.trim() || "").toUpperCase();
  const name  = [first, last].filter(Boolean).join(" ") || "Unknown";

  return {
    name,                              // "Jason KIM" → accounts.name
    firstName:    fields.firstName,
    lastName:     fields.lastName,
    englishName:  fields.englishName,
    originalName: fields.originalName,
    accountType:  "Student",
    email:        extra.email,
    phoneNumber:  extra.phone,
    ownerId:      extra.ownerId,
    status:       "Active",
  };
}
```

### 6.3 중복 확인 로직

Account 생성 전 반드시 중복 체크:

```typescript
// 중복 판단 기준: firstName + lastName + (email OR originalName)
async function checkDuplicateAccount(
  db: DrizzleDB,
  fields: SourceNameFields,
  email?: string
): Promise<Account | null> {
  const results = await db
    .select()
    .from(accounts)
    .where(
      and(
        eq(accounts.firstName, fields.firstName || ""),
        eq(accounts.lastName,  fields.lastName  || ""),
        eq(accounts.accountType, "Student"),
        or(
          email ? eq(accounts.email, email) : undefined,
          fields.originalName
            ? eq(accounts.originalName, fields.originalName)
            : undefined
        )
      )
    )
    .limit(1);

  return results[0] ?? null;
}
```

### PROMPT I: 변환 API 엔드포인트 (3종 통합)

```
server/src/routes/ 디렉토리에 아래 3개 API를 추가해.

공통 패턴:
- 소스 레코드에서 이름 필드 추출 (firstName, lastName, englishName, originalName)
- 중복 Account 확인: firstName + lastName + email 조합으로 기존 Student Account 검색
- 중복 없으면 → Account 생성 → Contact 생성 → 소스 레코드에 accountId, contactId 연결
- 중복 있으면 → 기존 Account 반환 + 소스 레코드 연결만 처리
- 응답: { account, contact, isDuplicate }

① POST /api/leads/:leadId/convert-to-account
   - leads 테이블: firstName, lastName, englishName, originalName, email(contact.emailAddress)
   - lead.accountId, lead.contactId 업데이트

② POST /api/applications/:appId/convert-to-account
   - applications 테이블: firstName, lastName, englishName, originalName, email
   - application.accountId, application.contactId 업데이트
   - application.applicationStatus → "converted"
   - application.convertedAt → NOW()

③ POST /api/camp-applications/:appId/convert-to-account
   - camp_applications 테이블: applicantFirstName, applicantLastName, applicantEnglishName, applicantOriginalName, applicantEmail
   - camp_applications.lead_id 경유로 account 연결 (Lead 먼저 생성 후 Account 변환)

서버 유틸리티:
server/src/utils/nameConverter.ts에 buildAccountFromName(), buildContactFromName() 함수 먼저 생성.
중복 확인 함수 checkDuplicateAccount()도 동일 파일에 작성.
```

### 3가지 진입 경로 × 이름 필드 비교

| 구분 | 테이블 | first_name | last_name | english_name | original_name | 비고 |
|------|--------|-----------|-----------|-------------|--------------|------|
| **Lead** | `leads` | `first_name` | `last_name` | `english_name` | `original_name` | 기존 `name` 필드 하위 호환 유지 |
| **Application (일반)** | `applications` | `first_name` | `last_name` | `english_name` | `original_name` | 처음부터 4분할 설계 |
| **Camp Application** | `camp_applications` | `applicant_first_name` | `applicant_last_name` | `applicant_english_name` | `applicant_original_name` | 기존 `applicant_name` 하위 호환 유지 |
| **Account** | `accounts` | `first_name` | `last_name` | `english_name` | `original_name` | `name` = fullName (영문 정식명) |
| **Contact** | `contacts` | `first_name` | `last_name` | `english_name` | `original_name` | 기존 `other_name` → `original_name` fallback |
