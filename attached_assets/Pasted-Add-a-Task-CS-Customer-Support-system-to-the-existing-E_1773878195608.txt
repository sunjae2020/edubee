Add a Task / CS (Customer Support) system to the existing Edubee Camp platform.
Do NOT modify existing tables or break existing functionality.

━━━ 1. DATABASE ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Add 3 new tables to the existing schema:

tasks:
  id, task_number (TSK-2026-0001), task_type ('internal'|'partner_work'|'cs_request'),
  category ('complaint'|'inquiry'|'request'|'document'|'payment'|'service'|'other'),
  source ('portal'|'email'|'phone'|'whatsapp'|'landing'),
  submitted_by (FK→users, nullable for public submissions),
  submitted_name, submitted_email, submitted_phone,
  assigned_to (FK→users), assigned_team,
  contract_id (FK→contracts, nullable), application_id (FK→applications, nullable),
  related_service_type ('institute'|'hotel'|'pickup'|'tour'|null),
  title, description,
  priority ('low'|'normal'|'high'|'urgent'),
  status ('open'|'in_progress'|'pending_response'|'resolved'|'closed'),
  visibility ('internal'|'shared'),
  due_date, first_response_at, resolved_at, sla_breached (bool),
  satisfaction_rating (1-5), satisfaction_comment, rated_at,
  created_at, updated_at

task_attachments:
  id, task_id (FK), file_path, file_name, file_type,
  uploaded_by (FK→users, nullable), created_at

task_comments:
  id, task_id (FK), author_id (FK→users, nullable),
  author_name (for public), content,
  is_internal (bool, default false), created_at

━━━ 2. BACKEND APIs ━━━━━━━━━━━━━━━━━━━━━━━━━

POST /api/public/tasks          ← no auth, landing page contact form
GET  /api/tasks                 ← auth, role-scoped
GET  /api/tasks/:id
POST /api/tasks                 ← auth
PUT  /api/tasks/:id
PATCH /api/tasks/:id/status
POST /api/tasks/:id/comments
POST /api/tasks/:id/attachments (multipart, reuse existing upload util)

Scope rules (use existing partnerScope pattern):
  SA/AD → all tasks
  camp_coordinator → tasks linked to own contracts/programs + own cs_requests
  education_agent → tasks linked to own applications/clients
  partner_* → tasks where assigned_team matches own role OR related_service_type matches
  parent_client → own submitted tasks only (cs_request, visibility=shared)

Task detail drawer (right slide-in):
