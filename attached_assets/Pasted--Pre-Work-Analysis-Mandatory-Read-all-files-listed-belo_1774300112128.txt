### [Accommodation] Add Overview Notes + New Host Family Tab
### ──────────────────────────────────────────

File: AccommodationDetail.tsx (or actual filename)

#### A. Notes Section at Bottom of Overview Tab
- Same implementation as Study Abroad
- entity_type = 'accommodation', entity_id = current record ID

#### B. New "Host Family" Tab

**[Key Design Principle]**
Host Family is not a separate table — it is a Supplier account in the accounts table.
- accommodation_mgt.provider_account_id → accounts.id
- account_type: 'supplier' (confirm actual value from DB)
- AP settlement history linked via account_ledger
- Previous placement history query:
  SELECT * FROM accommodation_mgt
  WHERE provider_account_id = [relevant account_id]

**Tab Layout — 4 Sections:**

[Section 1] Host Family Basic Info (from accounts table)
- Name (accounts.name)
- Contact and address (JOIN account_contacts)
- Account Type badge
- Registration date / Active status

[Section 2] Accommodation Details (from current accommodation_mgt record)
- Room type (room_type), Meal option (meal_included)
- Student weekly rate (weekly_rate) — AR
- Partner weekly cost (partner_weekly_cost) — AP
- Margin = weekly_rate − partner_weekly_cost (auto-calculated, displayed inline)
- Distance to school (distance_to_school)
- Check-in / Check-out dates
- Welfare check history (welfare_check_dates JSONB array displayed as a list)

[Section 3] Settlement Status (linked via account_ledger)
- Display account_ledger balance for this Host Family account
- Unpaid AP amount (contract_products WHERE ap_status IN ('pending', 'ready'))
- Paid AP amount
- Last settlement date

[Section 4] Previous Student Placement History (reverse lookup)
- Query:
  SELECT am.*, a.name AS student_name, c.contract_number
  FROM accommodation_mgt am
  JOIN accounts a ON am.student_account_id = a.id
  JOIN contracts c ON am.contract_id = c.id
  WHERE am.provider_account_id = [current provider_account_id]
  ORDER BY am.checkin_date DESC
- Columns: Student Name / Check-in / Check-out / Weekly Rate / Status badge / Contract link
- Distinguish: Currently placed (status: 'checked_in') vs Completed (status: 'checked_out')
- Footer aggregates: Total placements / Total cumulative AP paid
