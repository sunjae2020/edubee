### 쿼리 1 — 최신 payment_headers 확인

```sql
SELECT
  id,
  payment_ref,
  payment_type,
  total_amount,
  payment_info_id,
  received_from,
  paid_to,
  status,
  created_on
FROM payment_headers
ORDER BY created_on DESC
LIMIT 5;
```

**확인 포인트:** payment_headers에 데이터가 있는지 확인

---

### 쿼리 2 — transactions 최신 레코드 확인

### 쿼리 3 — credits 최신 레코드 확인


### 쿼리 5 — journal_entries 정상 유지 확인

```sql
SELECT
  id,
  entry_date,
  entry_type,
  debit_coa,
  credit_coa,
  amount,
  auto_generated,
  created_on
FROM journal_entries
ORDER BY created_on DESC
LIMIT 5;
```

**확인 포인트:** 기존 journal_entries 레코드가 브릿지 작업 후에도 정상 존재하는지 확인

---

### 쿼리 6 — 연결 무결성 확인 (credits ↔ transactions)

```sql
SELECT
  c.id AS credit_id,
  c.name AS credit_name,
  c.credit_amount,
  c.transaction_id,
  t.name AS transaction_name,
  t.payment_description
FROM credits c
LEFT JOIN transactions t ON c.transaction_id = t.id
WHERE c.created_on > NOW() - INTERVAL '7 days'
ORDER BY c.created_on DESC
LIMIT 5;
```

**확인 포인트:** credits.transaction_id → transactions.id 연결이 올바른지 확인

---

### 쿼리 7 — 연결 무결성 확인 (debits ↔ transactions)

```sql
SELECT
  d.id AS debit_id,
  d.name AS debit_name,
  d.debit_amount,
  d.transaction_id,
  t.name AS transaction_name,
  t.payment_description
FROM debits d
LEFT JOIN transactions t ON d.transaction_id = t.id
WHERE d.created_on > NOW() - INTERVAL '7 days'
ORDER BY d.created_on DESC
LIMIT 5;
```

**확인 포인트:** debits.transaction_id → transactions.id 연결이 올바른지 확인
