# 🐝 Edubee CRM — Schema Update: Task Type 분리
**작성일:** 2026-03-24  
**변경 유형:** tasks 테이블 컬럼 추가 (하위 호환 유지)  
**영향 범위:** tasks 테이블 1개 / 기존 데이터 보존

---

## 1. 변경 요약

| 구분 | 내용 |
|------|------|
| 변경 대상 | `tasks` 테이블 |
| 변경 종류 | 컬럼 추가 (기존 컬럼 변경 없음) |
| 추가 컬럼 | `task_type VARCHAR(20) NOT NULL DEFAULT 'internal'` |
| 추가 인덱스 | `idx_tasks_type ON tasks(task_type)` |
| 기존 데이터 처리 | `task_category = 'Support'` → `task_type = 'cs'` 자동 전환 |

---

## 2. task_type 값 정의

| 값 | 의미 | 사용 화면 |
|----|------|-----------|
| `internal` | 직원 내부 업무 (기본값) | Tasks 탭 |
| `cs` | 고객 응대 티켓 | CS 탭 |

## 4. 업데이트된 CREATE TABLE (SQL 파일 반영용)

> `edubee_crm_db_schema_final_20260321.sql` 의 tasks CREATE TABLE을 아래로 교체

```sql
-- 05. Task (직원 업무 항목 + 고객 응대)
CREATE TABLE IF NOT EXISTS tasks (
  id                    UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  name                  VARCHAR(255),
  manual_input          BOOLEAN     NOT NULL DEFAULT FALSE,
  task_type             VARCHAR(20) NOT NULL DEFAULT 'internal',  -- ★ 신규: internal | cs
  subject               VARCHAR(255),
  task_status           VARCHAR(50) NOT NULL,   -- New | In Progress | Completed | Cancelled
  task_category         VARCHAR(100),           -- Support | Follow-up | Admin | Visa
  start_date            TIMESTAMP,
  due_date              TIMESTAMP,
  end_date              TIMESTAMP,
  priority              VARCHAR(20),            -- Normal | High | Low | Urgent
  contact_id            UUID        REFERENCES contacts(id),
  account_id            UUID        REFERENCES accounts(id),
  contract_id           UUID        REFERENCES contracts(id),
  lead_id               UUID        REFERENCES leads(id),
  owner_id              UUID        NOT NULL REFERENCES users(id),
  description           TEXT,
  status                VARCHAR(20) NOT NULL DEFAULT 'Active',
  created_on            TIMESTAMP   NOT NULL DEFAULT NOW(),
  modified_on           TIMESTAMP   NOT NULL DEFAULT NOW()
);
```

## 5. 인덱스 섹션 추가 (SECTION 10 에 추가)

```sql
CREATE INDEX IF NOT EXISTS idx_tasks_type ON tasks(task_type);
```

기존 인덱스 두 줄 뒤에 추가:
```sql
CREATE INDEX IF NOT EXISTS idx_tasks_owner  ON tasks(owner_id);   -- 기존
CREATE INDEX IF NOT EXISTS idx_tasks_status ON tasks(task_status); -- 기존
CREATE INDEX IF NOT EXISTS idx_tasks_type   ON tasks(task_type);   -- ★ 신규
```

## 7. UI 구현 가이드 (프론트엔드 참고)

### 탭 구조 (Tasks 페이지)
```
[ Tasks | CS ]          ← 탭 필터 (task_type 기준)
  └─ Tasks: task_type = 'internal'
  └─ CS   : task_type = 'cs'
```

### API 엔드포인트 필터 예시
```
GET /api/tasks?task_type=internal   → 내부 업무
GET /api/tasks?task_type=cs         → 고객 응대
GET /api/tasks                      → 전체 (기존 동작 유지)
```

### 신규 Task 생성 시 기본값
- Tasks 탭에서 생성 → `task_type = 'internal'` 자동 설정
- CS 탭에서 생성 → `task_type = 'cs'` 자동 설정

## 8. Phase 2 확장 여지 (향후 참고)

현재 `task_type` 컬럼 하나로 충분하나, 아래 조건 발생 시 cs_tickets 독립 테이블로 분리 검토:

| 조건 | 설명 |
|------|------|
| 고객 포털 오픈 | 고객이 직접 CS 티켓 생성 |
| CS 전담팀 구성 | CS 전용 KPI/대시보드 필요 |
| SLA 자동 알림 | n시간 내 미응답 에스컬레이션 |
