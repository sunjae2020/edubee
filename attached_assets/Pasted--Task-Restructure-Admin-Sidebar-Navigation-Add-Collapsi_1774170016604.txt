## Task: Restructure Admin Sidebar Navigation + Add Collapsible Accordion

### Overview
This task has two parts, executed in order:
1. Reorganise the current admin sidebar from 45 scattered items into 10 logical categories
2. Add collapsible accordion behaviour to each category

This is a frontend-only change. Do NOT touch any backend files, routes, DB schema,
or authentication/permission logic.

---

### Pre-work (Required Before Any Changes)
1. Read the sidebar component file(s) in full
2. Identify how menu items are currently defined (config array, hardcoded JSX, etc.)
3. Identify how role-based visibility is currently implemented
4. Identify whether any collapse/expand state already exists
5. Report the full file list and your implementation plan — do NOT write any code
   until I explicitly say "go ahead"

---

### Safety Rules

**Must follow:**
- Modify one file at a time, run `npx tsc --noEmit` after each file
- Only fix what this task requires — do not refactor unrelated code

**Never do:**
- Delete or rename any existing URL paths
- Modify any page components
- Modify any backend files
- Modify authentication or role/permission logic
- Guess field names without reading the schema
- Modify multiple files in one go

## PART 2 — Collapsible Accordion

After completing Part 1, add accordion expand/collapse to each category.

---

### Default State on Page Load
- The category containing the **currently active route** is expanded
- All other categories are collapsed
- This applies on first visit, navigation, and page refresh

### Toggle Behaviour
- Clicking a category header toggles that category independently
- Other categories are NOT auto-closed when one opens (independent toggle)
- The active route's category is always expanded — even if localStorage says collapsed

### Persist State
- Storage key: `"edubee_sidebar_collapsed"`
- Storage value: JSON array of category keys that are currently **collapsed**
