Build the Finance backend on top of the existing Express + TypeScript server.

─── BUSINESS RULES (strictly follow) ───────────────────────────────

RULE 1 — Agent Commission is COST, not revenue:
  Net Camp Revenue = Total Fee − Agent Commission → cost center RC-CAMP
  Agent Commission → cost center CC-AGENT (payable)

RULE 2 — Primary Currency:
  All finance items use the contract's primary currency (default AUD).
  No automatic FX conversion. Manual adjustment only.

RULE 3 — Auto-generation:
  On contract creation → read linked Package → read Package Products
  → auto-generate contract_finance_items rows.
  All auto-generated items can be manually deleted / edited / added later.

RULE 4 — Commission calculation:
  If commissionType = 'percentage':
    Commission Amount = Total Fee × commissionRate / 100
  If commissionType = 'fixed':
    Commission Amount = commissionFixed
  Agent Invoice estimatedAmount = Total Fee − Commission Amount

─── PRODUCT TYPE → COST CENTER MAPPING ─────────────────────────────
  'institute'  → CC-INSTITUTE
  'hotel'      → CC-HOTEL
  'pickup'     → CC-PICKUP
  'tour'       → CC-TOUR
  'settlement' → CC-SETTLEMENT
  others       → CC-MISC

─── SERVICE: financeAutoGenerate(contractId) ────────────────────────
  1. Load contract + package + products + agent assignment
  2. Create RECEIVABLE items:
     a. Client Invoice → itemCategory: 'client_invoice'
        costCenter: RC-CAMP (if agent exists) or RC-DIRECT (no agent)
        estimatedAmount = Total Fee
     b. If agent linked → Agent Invoice → itemCategory: 'agent_invoice'
        costCenter: RC-CAMP
        commissionType, commissionRate/commissionFixed from package
        estimatedAmount = Total Fee − Commission Amount
  3. Create PAYABLE items from each Product in the Package:
     - One row per product, category and costCenter from mapping above
     - estimatedAmount from product price
     - linkedPartnerId from product's partner
  4. If agent linked → also create:
     CC-AGENT payable row (Agent Commission) 
     estimatedAmount = Commission Amount
  5. Set isAutoGenerated = true on all rows

─── API ENDPOINTS ───────────────────────────────────────────────────

POST   /api/contracts/:id/finance/generate
  → Trigger auto-generation (called automatically on contract creation)
  → Idempotent: skip if items already exist for this contract

GET    /api/contracts/:id/finance
  → Return all finance items grouped:
    { receivable: [...], payable: [...], summary: { ... } }
  → Summary object:
    { totalFee, agentCommission, netCampRevenue,
      totalPayable, grossProfit,
      costCenterBreakdown: [{ center, label, amount }],
      receivablePaid, receivablePending,
      payablePaid, payablePending }

POST   /api/contracts/:id/finance/items
  → Manually add a finance item (body: all fields)

PUT    /api/finance/items/:itemId
  → Edit item (amount, dueDate, notes, label, linkedPartnerId)

DELETE /api/finance/items/:itemId
  → Soft delete → set isDeleted = true

POST   /api/finance/items/:itemId/confirm-payment
  Body: { actualAmount, paidDate, referenceNo, paymentMethod, notes }
  Side effects (in order):
    1. Update finance item: status → 'paid', actualAmount, paidDate
    2. Create user_ledger entry:
         Receivable item → entryType: 'credit'
         Payable item    → entryType: 'debit'
         Set costCenter, amount, transactionDate, referenceType: 'invoice'
    3. Update linked invoice status → 'paid'
    4. If item is client_invoice or agent_invoice and fully paid:
         Auto-generate receipt record:
           receiptNumber: RCP-{YYYY}-{4-digit-seq}
           payerId: client or agent userId
           amount: actualAmount
           receiptDate: paidDate
           status: 'confirmed'
         Send receipt PDF via email to payer (Nodemailer)
         Create in-app notification:
           title: "Payment Confirmed — {receiptNumber}"
           type: receipt_generated

GET    /api/contracts/:id/finance/ledger
  → Return all user_ledger entries for this contract, sorted by transactionDate desc

─── CONTRACT CREATION TRIGGER ───────────────────────────────────────
In the existing POST /api/applications/:id/convert-contract endpoint,
after inserting the new contract record, immediately call:
  await financeAutoGenerate(newContractId)

─── ROLE ACCESS RULES ───────────────────────────────────────────────
SA, AD       → full access (view, edit, delete, confirm, ledger)
CC           → own contracts only (view, edit, confirm)
EA (Agent)   → own contracts only (view + download receipt only)
Partners     → no access to Finance tab
Client       → download own receipts only
