## 목적

Payment Header(신규 회계 시스템) 저장 시,
기존 Transactions + Credits/Debits 레코드를 자동으로 함께 생성하는
브릿지 로직을 추가한다.

결과: 사용자는 1회만 입력 → Xero/QuickBooks와 동일한 구조

---

## 파일 목록

```
STEP1_분석.md       ← 코드 변경 없음, 현황 파악
STEP2_브릿지구현.md  ← route 파일 1개에 로직 추가
STEP3_DB검증.md     ← SQL SELECT로 연동 확인
STEP4_E2E테스트.md  ← 실제 API 호출로 최종 확인
```

---

## Replit 사용 방법

1. Replit 열기
2. 상단 메뉴 → `New Task` (또는 AI 채팅창)
3. `STEP1_분석.md` 파일 내용을 붙여넣기 (또는 파일 업로드)
4. 실행 → 결과 확인
5. 통과 조건 충족 시 → 다음 Step 파일로 반복

---

## 통과 조건 요약

| Step | 통과 조건 |
|------|-----------|
| STEP 1 | 파일 경로 + 줄 번호 + 스키마 확인됨 |
| STEP 2 | tsc 오류 0개 + 서버 정상 기동 |
| STEP 3 | TRANSACTION- 형식 레코드 존재 + JOIN 정상 |
| STEP 4 | API 2회 호출 결과 모두 PASS |

---

## 중요 원칙

- DB 컬럼 삭제/변경 절대 금지
- 기존 journal_entries 로직 수정 금지
- 브릿지 실패해도 payment_header 저장 영향 없음 (try-catch)
- 각 Step은 이전 Step 통과 후에만 진행

---

## 문제 발생 시

STEP 1 결과가 예상과 다르거나
STEP 2~4에서 오류 발생 시

→ 결과를 claude.ai 채팅에 공유하면 프롬프트 조정 가능
