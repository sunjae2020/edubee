## Task 2 — Backend API

Create `/server/src/routes/applications.ts` with these endpoints:

```
GET    /api/applications           — list with filters (status, type)
GET    /api/applications/:id       — single record
POST   /api/applications           — create new + generate APP- ref
PATCH  /api/applications/:id       — update
DELETE /api/applications/:id       — soft delete (status = 'Inactive')

POST   /api/applications/:id/convert-to-quote   ← key endpoint
```

**convert-to-quote logic:**
1. Fetch the application record
2. If already `converted`, return an error
3. Generate `QTE-XXXXXXXX` ref number
4. Insert a new row into `quotes` table (status: `Draft`, note: summarised from application fields)
5. Update application: set `application_status = 'quoted'`, set `quote_id = newQuote.id`
6. Return `{ quote_id, quote_ref_number }`

Register the route in the main server file using the same pattern as existing routes.
