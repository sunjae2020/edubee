### STEP 1. 오류 전체 목록 재확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

npx tsc --noEmit 2>&1 실행 후 오류를 아래 형식으로 정리한다:

그룹 A — @workspace/db/schema import 오류
  → 없는 export를 import 하는 파일 목록
  → 없는 export 명 목록

그룹 B — 함수 반환 경로 오류
  → authenticate.ts, requireRole.ts, account-service-profiles.ts
  → Not all code paths return a value

그룹 C — JSX pragma 오류
  → generateReportPdf.tsx, PdfLogo.tsx

그룹 D — 기타 타입 오류
  → objectStorage.ts, chatbot.ts, accounting-transactions.ts

각 그룹별 오류 수와 내용을 보고 후
내가 "진행해"라고 할 때까지 수정하지 않는다.

### STEP 2. 그룹별 수정 (순서대로)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[그룹 A] @workspace/db/schema import 오류
수정 방식:
  ① lib/db/src/schema/index.ts 에서
     실제 export 되는 이름 목록 확인
  ② 각 파일의 잘못된 import 명을 실제 export 명으로 교체
  ③ 없는 테이블이면 import 줄 제거 + 사용 부분 주석 처리

[그룹 B] Not all code paths return a value
수정 방식:
  ① 함수 마지막에 return 없는 경로 확인
  ② 명시적 return (예: return next() / return res.status(403).json(...))
     또는 return undefined 추가

[그룹 C] JSX pragma 오류 (tsx 파일)
수정 방식:
  ① 파일 최상단에 React import 확인
     import React from 'react'; 없으면 추가
  ② tsconfig.json 에서 jsx 설정 확인
     "jsx": "react-jsx" 이면 React import 불필요
     → 오류 원인에 따라 판단

[그룹 D] 기타 타입 오류
각 파일별 오류 내용 확인 후 최소 수정:
  - objectStorage.ts: signed_url 타입 오류
    → as string 캐스팅 또는 옵셔널 체이닝
  - chatbot.ts: 타입 불일치
    → any 캐스팅 또는 정확한 타입 지정
  - accounting-transactions.ts: 타입 오류
    → 동일 패턴
