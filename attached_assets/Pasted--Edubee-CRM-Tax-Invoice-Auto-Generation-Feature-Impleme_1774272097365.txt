## 📐 Business Logic — Core Requirements

### 1. Remittance Method Classification (NET vs GROSS)

Based on the `payment_method` field in `contract_products` or `agent_commission_configs`:

```
payment_method = 'pre_deduct'  → NET Remittance
payment_method = 'invoice'     → GROSS Remittance (Full Remit)
```

---

### 2. NET Remittance (Pre-deduct) Flow

```
ap_status: 'ready' → Manager executes remittance
    │
    ├── Remit to school: (Total AR - Commission - GST*)
    │     * GST inclusion: branch by account type (see below)
    │
    ├── ap_status → 'paid'
    │
    └── [Triggered immediately & automatically]
          Generate Tax Invoice PDF
          → Send remittance confirmation + Tax Invoice to school email simultaneously
```

**GST Calculation Branch:**

```
School account type = 'domestic_aud' (Australian bank account)
  → Agency has ABN (organisations.has_abn = true)
  → GST = commission × 10%
  → School remittance = AR - commission - GST
  → Tax Invoice: itemise commission + GST

School account type = 'overseas_fx' (Overseas bank account)
  → GST-free (overseas transaction = Export)
  → GST = $0
  → School remittance = AR - commission
  → Tax Invoice: commission + "GST-free" notation
```

---

### 3. GROSS Remittance (Full Remit) Flow

```
ap_status: 'ready' → Manager executes remittance
    │
    ├── Remit full amount to school (commission NOT deducted)
    │
    ├── ap_status → 'paid'
    │
    ├── Commission → transferred to AR (Receivable)
    │     → contract_products.commission_ar_status = 'pending'
    │     → journal_entry: DR 1300 (AR—Commission Receivable) / CR 3100 (Commission—Tuition)
    │
    └── On course_start_date (scheduler)
          → Create notification for the assigned consultant:
            "Tax Invoice required: [Student Name] / [School Name] / Commission $XXX"
          → Consultant reviews and manually sends Tax Invoice
```

## 🗂️ DB Schema Changes

### New Table: `tax_invoices`

```sql
CREATE TABLE IF NOT EXISTS tax_invoices (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  invoice_ref           VARCHAR(50)   UNIQUE NOT NULL,  -- TINV-XXXXXXXX
  invoice_date          DATE          NOT NULL,
  invoice_type          VARCHAR(20)   NOT NULL,          -- 'net' | 'gross'
  contract_product_id   UUID          NOT NULL REFERENCES contract_products(id),
  contract_id           UUID          NOT NULL REFERENCES contracts(id),
  school_account_id     UUID          NOT NULL REFERENCES accounts(id),
  student_account_id    UUID          NOT NULL REFERENCES accounts(id),
  program_name          VARCHAR(255)  NOT NULL,
  student_name          VARCHAR(255)  NOT NULL,
  course_start_date     DATE,
  course_end_date       DATE,
  commission_amount     DECIMAL(12,2) NOT NULL DEFAULT 0,
  gst_amount            DECIMAL(12,2) NOT NULL DEFAULT 0,  -- 0 when GST-free
  total_amount          DECIMAL(12,2) NOT NULL DEFAULT 0,
  is_gst_free           BOOLEAN       NOT NULL DEFAULT FALSE,
  payment_header_id     UUID          REFERENCES payment_headers(id), -- linked AP remittance
  pdf_url               VARCHAR(500),
  sent_at               TIMESTAMP,
  sent_to_email         VARCHAR(255),
  status                VARCHAR(20)   NOT NULL DEFAULT 'draft',
  -- 'draft' | 'sent' | 'paid'  (GROSS: updated when commission received)
  due_date              DATE,         -- applicable for GROSS invoices
  paid_at               TIMESTAMP,    -- date commission received (GROSS)
  created_by            UUID          NOT NULL REFERENCES users(id),
  created_on            TIMESTAMP     NOT NULL DEFAULT NOW(),
  modified_on           TIMESTAMP     NOT NULL DEFAULT NOW()
);
```

### Existing Table Field Additions

```sql
-- contract_products: track GROSS commission AR
ALTER TABLE contract_products ADD COLUMN IF NOT EXISTS
  remittance_method    VARCHAR(20),   -- 'net' | 'gross'
  commission_ar_status VARCHAR(20),   -- NULL | 'pending' | 'invoiced' | 'received'
  commission_amount    DECIMAL(12,2),
  gst_amount           DECIMAL(12,2),
  is_gst_free          BOOLEAN DEFAULT FALSE;

-- organisations (agency tenant): add ABN fields
ALTER TABLE organisations ADD COLUMN IF NOT EXISTS
  has_abn     BOOLEAN       DEFAULT FALSE,
  abn         VARCHAR(20),
  abn_name    VARCHAR(255); -- Registered business name for ABN

-- accounts (add school bank account type)
ALTER TABLE accounts ADD COLUMN IF NOT EXISTS
  bank_account_type  VARCHAR(20);
  -- 'domestic_aud' | 'overseas_fx'
```

## 📊 Journal Entry Handling

### NET Remittance — Journal Entries

```
Remittance execution (existing):
  DR  2100  AP — Student Tuition Payable    $2,000
  CR  1200  Trust Account                   $2,000

Commission separation (NEW addition):
  DR  1200  Trust Account                     $200   (commission retained in Trust)
  CR  3100  Commission — Tuition              $200
  ── If GST applies ──
  DR  1200  Trust Account                      $20   (GST retained in Trust)
  CR  GST Collected  (confirm CoA code)         $20
```

### GROSS Remittance — Commission AR Transfer (NEW)

```
After full remittance to school:
  DR  1300  AR — Commission Receivable        $200
  CR  3100  Commission — Tuition              $200
  ── If GST applies ──
  DR  1300  AR — Commission Receivable         $20
  CR  GST Collected                            $20

When commission is actually received from school:
  DR  1100  Cash & Bank — Operating           $220
  CR  1300  AR — Commission Receivable        $220
```
